## 🔷 Inheritance in Dart
Inheritance means a class can use the properties and methods of another class.

## Key Idea
* Parent Class (Superclass) → gives properties & methods
* Child Class (Subclass) → receives and uses them

## How It Works
* Child class automatically gets:
  * Variables (properties)
  * Functions (methods)
* No need to rewrite code → saves time

```dart
class Father{
  String name = "Kabir";
  String property = "Flat";
  String incomesource = "Rent";
}

class Son extends Father{
  
}

void main(){
  Son azmir = Son();
  print(azmir.name);
  print(azmir.property);
  print(azmir.incomesource);
}
```

## Output

```sql
Kabir
Flat
Rent
```

### What’s happening here?
## Parent Class

```dart
class Father {
  String name = "Kabir";
  String property = "Flat";
  String incomesource = "Rent";
}
```

* Father class has:
  * name
  * property
  * income source
   
## Child Class

```dart
class Son extends Father {

}
```

* Son class inherits everything from Father using extends
* Even though Son is empty, it still gets all properties

## Main Function

```dart
void main() {
  Son azmir = Son();
  print(azmir.name);
  print(azmir.property);
  print(azmir.incomesource);
}
```

* You created an object azmir of Son
* But you are accessing:
  * name
  * property
  * incomesource
* These are not inside Son, they come from Father class

## Output
```sql
Kabir
Flat
Rent
```

## Key Understanding
* Son automatically got all data from Father
* No need to rewrite anything
* This is called pure inheritance
  
## Simple Real-Life Idea

* Son inherits from Father
So he gets:
  * Name
  * Property
  * Income source

## 🔷 Variable Override

```dart
class Father{
  String name = "Kabir";
  String property = "Flat";
  String incomeSource = "Rent";
}

class Son extends Father{
  @override
  String incomeSource = "Job";
  String incomeSource1 = "App Development";
}

void main(){
  Son azmir = Son();
  print(azmir.name);
  print(azmir.property);
  print(azmir.incomeSource);

  Son anam = Son();
  print(anam.incomeSource1);
}
```

### What just we did here

## Parent class

```dart
class Father {
  String name = "Kabir";
  String property = "Flat";
  String incomeSource = "Rent";
}
```

* Defines default values

## Child class

```dart
class Son extends Father {
  @override
  String incomeSource = "Job";          // overridden
  String incomeSource1 = "App Development"; // new property
}
```

## Two important things here:

1. Overriding
* incomeSource was "Rent" in Father
* Now changed to "Job" in Son
  * Same type (String) → correct overriding

2. New property
* incomeSource1 is only in Son
* Father doesn’t have it

  
## Main function

```dart
Son azmir = Son();
print(azmir.name);         // from Father
print(azmir.property);     // from Father
print(azmir.incomeSource); // overridden in Son

Son anam = Son();
print(anam.incomeSource1); // Son's own property
```

```sql
📌 Output
Kabir
Flat
Job
App Development
```

## Important Understanding
* name, property → inherited from Father
* incomeSource → overridden in Son
* incomeSource1 → new in Son


## 🔷 Method Overriding

```dart
class Father{
  String name = "Kabir";
  String incomeSource = "Rent";

  void property(){
    print("Land");
  }
}

class Son extends Father{
  @override
  String incomeSource = "Job";
  String incomeSource1 = "App Development";

  @override
  property(){
    print("Flat");
  }

}

void main(){
  Son azmir = Son();
  print(azmir.name);
  print(azmir.incomeSource);
  azmir.property();

  Son anam = Son();
  print(anam.incomeSource1);
}
```
### Final Understanding

* Inheritance → reuse code
* Override variable → change data
* Override method → change behavior
* Add new variable → extend functionality

## 🔷 Constructor

A constructor is a special function in a class that runs automatically when you create an object.

## Example of a simple constructor

```dart
class Person {
  String name;
  int age;

  // Constructor
  Person(String n, int a) {
    name = n;
    age = a;
  }
}

void main() {
  Person p1 = Person("Azmir", 22); // Constructor runs here
  Person p2 = Person("Rahim", 25);

  print(p1.name); // Azmir
  print(p1.age);  // 22
  print(p2.name); // Rahim
  print(p2.age);  // 25
}
```
What happens step by step:

1. Person p1 = Person("Azmir", 22);
  * Dart calls the constructor Person(String n, int a) automatically.
3. Constructor sets name = n and age = a.
4. Object p1 is ready with its values.

---

## Why constructors are useful

Without constructors, every object would need manual assignment:

```dart
Person p1 = Person();
p1.name = "Azmir";
p1.age = 22;
// With constructors, it’s automatic:
Person p1 = Person("Azmir", 22);
```
* Cleaner, faster, less error-prone

---

## Constructor + Inheritance

When a child class inherits from a parent class, the parent constructor must run first to set up parent properties.

```dart
class Student{
  String name;
  int age;
  Student(this.name,this.age);
}

class One extends Student{
  int student;
  One(super.name, super.age, this.student){
    print("Student 100");
  }
}

void main(){
  One obj = One("Nahim", 18, 58);
  print("Name: ${obj.name}");
  print("Age: ${obj.age}");
  print("Student: ${obj.student}");
}

Output:

```sql
Student 100
Name: Nahim
Age: 18
Student: 58
```

* super(n) → calls the Father’s constructor first
* Then Son’s constructor runs

💡 Rule: Parent part of object must be initialized before child part.
