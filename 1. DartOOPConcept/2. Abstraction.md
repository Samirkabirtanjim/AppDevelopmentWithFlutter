```dart
abstract class Animal{
  String name;
  Animal(this.name);

  void eat(){
    print("${name} can Eat");
  }
  void makeSound(){
    print("${name} can make sound");
  }
}

class Cat extends Animal{
  String fly;

  Cat(super.name, this.fly);
}

void main(){
  Cat jerry = Cat("Jerry", "Can't fly");
  print("Cat Name: ${jerry.name}");
  print("Cat Fly: ${jerry.fly}");
  jerry.eat();
  jerry.makeSound();
}
```

## 🔷 1. What your code does
📌 Abstract class?

```dart
abstract class Animal {
  String name;
  Animal(this.name);

  void eat() {
    print("${name} can Eat");
  }

  void makeSound() {
    print("${name} can make sound");
  }
}
```

* Animal is abstract → cannot create objects directly
* But it can have variables and methods
* Constructor Animal(this.name) is used to initialize name
* eat() and makeSound() are normal methods

## Child class
```dart
class Cat extends Animal {
  String fly;

  Cat(super.name, this.fly);
}
```

* Cat inherits from Animal → gets name, eat(), makeSound()
* super.name → calls parent constructor automatically
* fly → new property for Cat

## Main Function

```dart
Cat jerry = Cat("Jerry", "Can't fly");
print("Cat Name: ${jerry.name}");
print("Cat Fly: ${jerry.fly}");
jerry.eat();
jerry.makeSound();
```

## 🔷 Output
```sql
Cat Name: Jerry
Cat Fly: Can't fly
Jerry can Eat
Jerry can make sound
```

## 🔷 Important Notes

1. Abstract class
* Cannot do: Animal a = Animal("Dog"); ❌
* Can force children to implement certain methods if we declare abstract methods
2. super.name
* Calls parent constructor to initialize name
* Same as writing Cat(String name, this.fly) : super(name);
3. Inheritance works perfectly
* Cat can reuse parent methods eat() and makeSound()
4. Child adds own property
* fly is only in Cat

## 🔹 What does implements mean?

When a class uses implements, it must provide implementations for all methods declared in the interface.
This example shows how to use `implements` in Dart.

```dart
class Person{

  void name(){

  }
  void age(){

  }
  void weight(){

  }
  void height(){

  }
}

class student implements Person{
  @override
  void age() {
    print("Age is 12");
  }

  @override
  void height() {
    print("MR.X height 30kg");
  }

  @override
  void name() {
    print("Name is MR.X");
  }

  @override
  void weight() {
    print("MR.X weight 4.1");
  }
}

void main(){
  student x = student();
  x.name();
  x.age();
  x.height();
  x.weight();
}

```

```dart
class student implements Person
```

This is correct because:

* Person is being used like an interface (contract)
* You are forcing student to define all methods
