# Difference Between `implements` and `extends` in Dart

This note explains the difference between `implements` and `extends` in Dart in a simple way with examples.

---

## Main Difference

- `extends` is used for **inheritance**
- `implements` is used for **following a contract**

In simple words:

- `extends` means a class **inherits code**
- `implements` means a class **must write the code itself**

---

## 1. Using `extends`

When a class uses `extends`, it gets the properties and methods of the parent class.

### Example

```dart
class GovRules {
  void takeOrder() {
    print("Taking order");
  }

  void servingFood() {
    print("Serving food");
  }

  void billGenerate() {
    print("Generating bill");
  }
}

class KFC extends GovRules {
  @override
  void takeOrder() {
    print("KFC Taking Order");
  }
}

void main() {
  KFC kfc = KFC();

  kfc.takeOrder();
  kfc.servingFood();
  kfc.billGenerate();
}
```

### Output

```text
KFC Taking Order
Serving food
Generating bill
```

### Explanation

Here:

```dart
class KFC extends GovRules
```

means:

- `KFC` is inheriting from `GovRules`
- `KFC` automatically gets all methods from `GovRules`
- `KFC` can override only the methods it wants to change

In this example:

- `takeOrder()` is overridden by `KFC`
- `servingFood()` is inherited from `GovRules`
- `billGenerate()` is inherited from `GovRules`

So `extends` helps with **code reuse**

---

## 2. Using `implements`

When a class uses `implements`, it does not inherit method bodies.

It only follows the structure or rule of another class.

### Example

```dart
abstract class GovRules {
  void takeOrder();
  void servingFood();
  void billGenerate();
}

class KFC implements GovRules {
  @override
  void takeOrder() {
    print("KFC Taking Order");
  }

  @override
  void servingFood() {
    print("KFC Serving Food");
  }

  @override
  void billGenerate() {
    print("KFC Generate Bills");
  }
}

void main() {
  KFC kfc = KFC();

  kfc.takeOrder();
  kfc.servingFood();
  kfc.billGenerate();
}
```

### Output

```text
KFC Taking Order
KFC Serving Food
KFC Generate Bills
```

### Explanation

Here:

```dart
class KFC implements GovRules
```

means:

- `KFC` must follow all rules from `GovRules`
- `KFC` must define every method
- `KFC` does not inherit any method body from `GovRules`

So when using `implements`, the class must write all methods by itself.

This is useful when different classes need the same method names, but each class should have its own logic.

---

## 3. Side by Side Comparison

### `extends`

```dart
class Parent {
  void show() {
    print("Parent show");
  }
}

class Child extends Parent {}

void main() {
  Child child = Child();
  child.show();
}
```

### Output

```text
Parent show
```

### `implements`

```dart
class Parent {
  void show() {
    print("Parent show");
  }
}

class Child implements Parent {
  @override
  void show() {
    print("Child show");
  }
}

void main() {
  Child child = Child();
  child.show();
}
```

### Output

```text
Child show
```

### Key Point

With `extends`, the child can use the parent method directly.

With `implements`, the child must rewrite the method.

## 4. Restaurant Example With `implements`

This matches your original idea.

```dart
abstract class GovRules {
  void takeOrder();
  void servingFood();
  void billGenerate();
}

class KFC implements GovRules {
  @override
  void takeOrder() {
    print("KFC Taking Order");
  }

  @override
  void servingFood() {
    print("KFC Serving Food");
  }

  @override
  void billGenerate() {
    print("KFC Generate Bills");
  }
}

class BFC implements GovRules {
  @override
  void takeOrder() {
    print("BFC Taking Order");
  }

  @override
  void servingFood() {
    print("BFC Serving Food");
  }

  @override
  void billGenerate() {
    print("BFC Generate Bills");
  }
}

void main() {
  GovRules mrx = KFC();
  mrx.takeOrder();
  mrx.servingFood();
  mrx.billGenerate();

  print("==================");

  GovRules mry = BFC();
  mry.takeOrder();
  mry.servingFood();
  mry.billGenerate();
}
```

### Output

```text
KFC Taking Order
KFC Serving Food
KFC Generate Bills
==================
BFC Taking Order
BFC Serving Food
BFC Generate Bills
```

### Explanation

Here both `KFC` and `BFC` follow the same `GovRules`, but each class provides its own implementation.

That is why `implements` is a good choice here.

## 5. Restaurant Example With `extends`

Now look at the same idea using `extends`.

```dart
class GovRules {
  void takeOrder() {
    print("Taking order");
  }

  void servingFood() {
    print("Serving food");
  }

  void billGenerate() {
    print("Generating bill");
  }
}

class KFC extends GovRules {
  @override
  void takeOrder() {
    print("KFC Taking Order");
  }
}

class BFC extends GovRules {
  @override
  void takeOrder() {
    print("BFC Taking Order");
  }
}

void main() {
  KFC kfc = KFC();
  kfc.takeOrder();
  kfc.servingFood();
  kfc.billGenerate();

  print("==================");

  BFC bfc = BFC();
  bfc.takeOrder();
  bfc.servingFood();
  bfc.billGenerate();
}
```

### Output

```text
KFC Taking Order
Serving food
Generating bill
==================
BFC Taking Order
Serving food
Generating bill
```

### Explanation

Here `KFC` and `BFC` inherit the common behavior from `GovRules`.

Only `takeOrder()` is changed.

That makes `extends` useful when many classes share common code.

---

## 6. Easy Memory Trick

Remember it like this:

### `extends`

- child class gets parent code
- used for inheritance
- used for code reuse

### `implements`

- class follows rules only
- no code is inherited
- all methods must be written again

---

## 7. When to Use Which

Use `extends` when:

- you want to reuse code from a parent class
- classes share common behavior
- only a few methods need to change

Use `implements` when:

- you want classes to follow the same structure
- each class should have its own implementation
- you want contract based design

---

## 8. Final Summary

### `extends`

```dart
class Child extends Parent
```

means:

> Child inherits code from Parent

### `implements`

```dart
class Child implements Parent
```

means:

> Child follows Parent rules but writes its own code

---

## 9. Conclusion

The difference between `extends` and `implements` is very important in Dart.

- `extends` gives inheritance
- `implements` gives structure only

So the short idea is:

- use `extends` for shared code
- use `implements` for shared rules

In your restaurant example, `implements` is a better choice when each restaurant should define its own way of taking orders, serving food, and generating bills.
