# Dart Abstraction and Polymorphism Example

This project explains how **abstraction** and **polymorphism** work in Dart using an `abstract class` and the `implements` keyword.

---

## Source Code

```dart
abstract class GovRules {
  void takeOrder();
  void servingfood();
  void billgenerate();
}

class KFC implements GovRules {
  @override
  void billgenerate() {
    print("KFC Generate Bills");
  }

  @override
  void servingfood() {
    print("KFC Serving Food");
  }

  @override
  void takeOrder() {
    print("KFC Taking Order");
  }
}

class BFC implements GovRules {
  @override
  void billgenerate() {
    print("BFC Generate Bills");
  }

  @override
  void servingfood() {
    print("BFC Serving Food");
  }

  @override
  void takeOrder() {
    print("BFC Taking Order");
  }
}

void main() {
  GovRules MRX = KFC();
  MRX.takeOrder();
  MRX.servingfood();
  MRX.billgenerate();

  print("==================");

  GovRules MRY = BFC();
  MRY.takeOrder();
  MRY.servingfood();
  MRY.billgenerate();
}
```

---

## What is `abstract class`?

```dart
abstract class GovRules {
  void takeOrder();
  void servingfood();
  void billgenerate();
}
```

`GovRules` is an **abstract class**.

It works like a common rule or structure for all restaurant classes.

It says that every restaurant must have these methods:

- `takeOrder()`
- `servingfood()`
- `billgenerate()`

But it does not provide the method body.

So this class only defines **what must be done**, not **how it is done**.

---

## What does `implements` mean?

```dart
class KFC implements GovRules
class BFC implements GovRules
```

The `implements` keyword means that the class must follow all rules from `GovRules`.

So both `KFC` and `BFC` must define:

- `takeOrder()`
- `servingfood()`
- `billgenerate()`

That is why both classes provide their own method implementations.

---

## Important Line

```dart
GovRules MRX = KFC();
```

This means:

- `KFC()` creates an object of the `KFC` class
- `MRX` stores that object
- the variable type is `GovRules`

So `MRX` is a **reference of type `GovRules`** that points to a **`KFC` object**

This is valid because `KFC` implements `GovRules`.

The same idea is used here:

```dart
GovRules MRY = BFC();
```

Now `MRY` is a `GovRules` reference that points to a `BFC` object.

---

## Why is this useful?

This allows us to use a common type for different classes.

Example:

```dart
GovRules MRX = KFC();
GovRules MRY = BFC();
```

Both are `GovRules`, but they point to different real objects.

When methods are called, Dart runs the correct version based on the actual object.

```dart
MRX.takeOrder(); // KFC version
MRY.takeOrder(); // BFC version
```

This is called **polymorphism**.

---

## Program Flow

### 1. Create `KFC` object

```dart
GovRules MRX = KFC();
```

### 2. Call methods on `MRX`

```dart
MRX.takeOrder();
MRX.servingfood();
MRX.billgenerate();
```

Output:

```text
KFC Taking Order
KFC Serving Food
KFC Generate Bills
```

### 3. Print separator

```dart
print("==================");
```

### 4. Create `BFC` object

```dart
GovRules MRY = BFC();
```

### 5. Call methods on `MRY`

```dart
MRY.takeOrder();
MRY.servingfood();
MRY.billgenerate();
```

Output:

```text
BFC Taking Order
BFC Serving Food
BFC Generate Bills
```

---

## OOP Concepts Used

### 1. Abstraction

`GovRules` defines a common structure for all restaurant classes.

### 2. Polymorphism

A `GovRules` type variable can hold different objects like `KFC` and `BFC`.

At runtime, Dart calls the correct method based on the actual object.

---

## Key Point

```dart
GovRules MRX = KFC();
```

This means:

> A parent type reference is holding a child class object.

That is the main idea behind polymorphism in this example.

---

## Better Dart Naming Style

Your code is correct, but the method names are not in common Dart style.

A cleaner version would be:

```dart
abstract class GovRules {
  void takeOrder();
  void serveFood();
  void generateBill();
}
```

This is easier to read and follows standard Dart naming conventions.

---

## Conclusion

This example is a simple way to understand how Dart uses:

- `abstract class`
- `implements`
- abstraction
- polymorphism

It shows that different classes can follow the same structure but provide their own behavior.
